## [](#网络层概述)网络层概述

### [](#网络层的核心功能)网络层的核心功能

网络层是OSI模型的第三层，主要负责将数据从源主机传送到目的主机，核心功能包括：

- **路由选择**：为分组选择合适的传输路径
- **分组转发**：将分组从入接口转发到出接口
- **拥塞控制**：防止网络过载
- **异构网络互联**：实现不同网络的互联互通

### [](#数据平面与控制平面)数据平面与控制平面

- **数据平面**：负责实际的分组转发，处理每一个经过路由器的分组
- **控制平面**：负责路由选择和路由表的维护，决定分组的转发路径

## [](#IP协议)IP协议

### [](#IPv4地址)IPv4地址

IPv4地址是32位（4字节）的逻辑地址，由**网络号**和**主机号**组成。

**分类编址**：

| 类别 | 首字节范围 | 网络号位数 | 最大网络数 | 最大主机数 |
|------|-----------|-----------|-----------|-----------|
| A类 | 1-126 | 8位 | 126 | 16777214 |
| B类 | 128-191 | 16位 | 16384 | 65534 |
| C类 | 192-223 | 24位 | 2097152 | 254 |
| D类 | 224-239 | - | 组播地址 | - |
| E类 | 240-255 | - | 保留地址 | - |

**特殊IP地址**：

| 地址 | 含义 |
|------|------|
| 0.0.0.0 | 本机（启动时使用） |
| 127.0.0.1 | 环回地址 |
| 255.255.255.255 | 广播地址 |
| 10.x.x.x | A类私有地址 |
| 172.16.x.x - 172.31.x.x | B类私有地址 |
| 192.168.x.x | C类私有地址 |

### [](#子网划分与子网掩码)子网划分与子网掩码

**子网掩码**：用于区分IP地址中的网络部分和主机部分

- A类默认子网掩码：255.0.0.0
- B类默认子网掩码：255.255.0.0
- C类默认子网掩码：255.255.255.0

**子网划分步骤**：
1. 确定需要的子网数
2. 计算需要的子网位数n（2^n ≥ 子网数）
3. 从主机位中借n位作为子网位
4. 计算每个子网的主机数（2^剩余主机位 - 2）

**面试题**：某C类网络192.168.1.0/24需要划分为4个子网，每个子网最多多少台主机？

- 借2位作为子网位（2^2=4个子网）
- 剩余6位作为主机位（2^6-2=62台主机）
- 子网掩码：255.255.255.192（/26）

### [](#CIDR无类域间路由)CIDR无类域间路由

CIDR打破了传统分类编址的限制：
- 使用任意长度的网络前缀
- 支持路由聚合（超网）
- 表示方式：a.b.c.d/n（n为网络前缀位数）

**路由聚合示例**：
- 192.168.0.0/24 + 192.168.1.0/24 → 192.168.0.0/23
- 减少路由表条目，提高转发效率

### [](#IPv4首部格式)IPv4首部格式

| 字段 | 长度 | 说明 |
|------|------|------|
| 版本 | 4bit | IPv4为4 |
| 首部长度 | 4bit | 以4字节为单位，最小5 |
| 区分服务 | 8bit | 服务质量 |
| 总长度 | 16bit | 整个数据报长度（最大65535字节） |
| 标识 | 16bit | 唯一标识数据报 |
| 标志 | 3bit | DF（禁止分片）、MF（还有分片） |
| 片偏移 | 13bit | 以8字节为单位 |
| 生存时间TTL | 8bit | 每经过一个路由器减1 |
| 协议 | 8bit | 上层协议（6=TCP，17=UDP） |
| 首部校验和 | 16bit | 仅校验首部 |
| 源地址 | 32bit | 发送方IP |
| 目的地址 | 32bit | 接收方IP |

### [](#IP分片)IP分片

- 当数据报长度超过MTU时需要分片
- 以太网MTU = 1500字节
- 分片时，数据部分按8字节整数倍划分
- **面试重点**：每个分片都有相同的标识字段，MF标志位=1表示后面还有分片，最后一个分片MF=0

### [](#NAT网络地址转换)NAT网络地址转换

- 将私有IP地址转换为公有IP地址
- 解决IPv4地址不足的问题
- 分为：静态NAT、动态NAT、NAPT（端口地址转换）
- **NAT穿透问题**：外部主机无法主动访问NAT后面的主机

## [](#ARP协议)ARP协议

### [](#ARP工作原理)ARP工作原理

地址解析协议（Address Resolution Protocol）将IP地址映射为MAC地址：

1. 发送方查看本地ARP缓存
2. 缓存中没有则广播ARP请求（目的MAC为FF:FF:FF:FF:FF:FF）
3. 目标主机单播ARP响应
4. 发送方将映射关系存入ARP缓存

### [](#ARP欺骗)ARP欺骗

- 攻击者发送伪造的ARP响应
- 使受害者将攻击者的MAC地址与网关IP关联
- 实现中间人攻击
- 防御：静态ARP绑定、ARP防火墙

### [](#ARP与RARP)ARP与RARP

- **ARP**：IP → MAC
- **RARP**：MAC → IP（已很少使用，被DHCP替代）
- **代理ARP**：路由器代替目标主机应答ARP请求

## [](#ICMP协议)ICMP协议

### [](#ICMP概述)ICMP概述

互联网控制报文协议（Internet Control Message Protocol）：
- 用于在IP主机和路由器之间传递控制消息
- ICMP报文封装在IP数据报中（协议字段为1）
- 不用于传输数据，只用于报告错误和状态

### [](#ICMP报文类型)ICMP报文类型

**ICMP差错报告报文**：
- 终点不可达（类型3）
- 时间超过（类型11，TTL为0）
- 参数问题（类型12）
- 改变路由/重定向（类型5）

**ICMP询问报文**：
- 回送请求/应答（类型8/0）——ping命令使用
- 时间戳请求/应答（类型13/14）

**不发送ICMP差错报文的情况**：
- 对ICMP差错报文不再发送ICMP差错报文
- 对第一个分片的数据报片之外的后续数据报片不发送
- 对多播地址不发送
- 对特殊地址（127.0.0.1、0.0.0.0）不发送

### [](#ping与traceroute)ping与traceroute

- **ping**：使用ICMP回送请求/应答报文测试连通性
- **traceroute**：利用TTL递增的ICMP时间超过报文追踪路由路径

## [](#路由选择协议)路由选择协议

### [](#内部网关协议RIP)内部网关协议——RIP

路由信息协议（Routing Information Protocol）：
- 使用**距离向量算法**
- 度量：跳数（最大15跳，16跳为不可达）
- 每30秒向邻居广播整个路由表
- 适合小型网络

**RIP的三个要点**：
1. 仅和相邻路由器交换信息
2. 交换的是当前路由器的完整路由表
3. 按固定时间间隔交换信息

**RIP的缺点**：
- 收敛慢
- 最大跳数限制
- 存在路由环路问题（通过水平分割、毒性逆转解决）

### [](#内部网关协议OSPF)内部网关协议——OSPF

开放最短路径优先（Open Shortest Path First）：
- 使用**链路状态算法**（Dijkstra算法）
- 度量：代价（可配置，通常与带宽相关）
- 向全网泛洪链路状态信息
- 适合大型网络

**OSPF的区域划分**：
- 骨干区域（Area 0）：所有非骨干区域必须与骨干区域相连
- 非骨干区域：可以进一步划分

**OSPF的五种报文**：

| 类型 | 名称 | 功能 |
|------|------|------|
| 1 | Hello | 发现和维护邻居关系 |
| 2 | DBD | 数据库描述 |
| 3 | LSR | 链路状态请求 |
| 4 | LSU | 链路状态更新 |
| 5 | LSAck | 链路状态确认 |

**RIP与OSPF对比**：

| 特性 | RIP | OSPF |
|------|-----|------|
| 算法 | 距离向量 | 链路状态 |
| 度量 | 跳数 | 代价 |
| 收敛速度 | 慢 | 快 |
| 网络规模 | 小型 | 大型 |
| 交换内容 | 完整路由表 | 链路状态 |
| 路由环路 | 可能出现 | 不会出现 |

### [](#外部网关协议BGP)外部网关协议——BGP

边界网关协议（Border Gateway Protocol）：
- 用于不同自治系统（AS）之间的路由选择
- 使用**路径向量算法**
- 交换的是到达各目的网络的路径信息
- BGP发言人之间建立TCP连接（端口179）

**BGP的特点**：
- 不追求最优路由，而是寻找能够到达目的地的较好路由
- 支持丰富的路由策略
- 是互联网的核心路由协议

## [](#IP组播)IP组播

### [](#组播概述)组播概述

- 一对多通信方式
- 使用D类IP地址（224.0.0.0 - 239.255.255.255）
- 组播组是开放的，主机可以随时加入或离开
- 使用IGMP协议管理组播组成员关系

### [](#IGMP协议)IGMP协议

互联网组管理协议（Internet Group Management Protocol）：
- 用于主机和路由器之间交换组播组成员信息
- 路由器定期查询组播组成员
- 主机通过成员报告加入组播组

## [](#移动IP)移动IP

### [](#移动IP的基本概念)移动IP的基本概念

- **归属网络**：移动节点的永久居所
- **归属代理**：归属网络中的路由器
- **外地网络**：移动节点当前所在网络
- **外地代理**：外地网络中的路由器
- **转交地址**：移动节点在外地网络获得的临时地址

### [](#移动IP通信过程)移动IP通信过程

1. 移动节点到达外地网络后获取转交地址
2. 向归属代理注册转交地址
3. 通信对端将数据发送到归属地址
4. 归属代理通过隧道将数据转发到转交地址
5. 移动节点直接向通信对端发送数据

## [](#IPv6)IPv6

### [](#IPv6的主要变化)IPv6的主要变化

- 地址从32位扩展到**128位**
- 首部格式简化，固定40字节
- 取消了首部校验和
- 取消了分片/重组相关字段（只在源端分片）
- 支持即插即用（自动配置地址）
- 内置安全性（IPSec）

### [](#IPv6地址表示)IPv6地址表示

- 128位，用冒号分隔的8组十六进制数
- 例如：2001:0db8:85a3:0000:0000:8a2e:0370:7334
- 简化规则：每组前导零可省略，连续全零组可用::替代（只能用一次）

### [](#IPv4到IPv6的过渡)IPv4到IPv6的过渡

- **双栈策略**：设备同时运行IPv4和IPv6协议栈
- **隧道策略**：将IPv6数据报封装在IPv4数据报中传输

## [](#网络层设备)网络层设备

### [](#路由器)路由器

- 工作在网络层
- 根据路由表转发分组
- 隔离广播域
- 支持多种路由协议

**路由器的组成**：
- 输入端口：物理层接收、链路层解封装、网络层查找路由表
- 交换结构：将分组从输入端口转移到输出端口
- 输出端口：缓存管理、链路层封装、物理层发送
- 路由处理器：运行路由算法，构建路由表

### [](#路由聚合)路由聚合

路由聚合（路由汇总）将多条路由合并为一条：
- 减少路由表条目
- 降低路由更新开销
- 提高转发效率

## [](#总结)总结

网络层是计算机网络的核心层，需要重点掌握：

1. **IP地址**：分类编址、子网划分、CIDR
2. **IP协议**：首部格式、分片机制、NAT
3. **ARP协议**：IP与MAC地址映射
4. **ICMP协议**：差错报告、ping/traceroute
5. **路由协议**：RIP距离向量、OSPF链路状态、BGP路径向量
6. **IPv6**：128位地址、过渡策略
