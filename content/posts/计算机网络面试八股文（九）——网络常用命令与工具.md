## [](#概述)概述

面试中经常会被问到网络排错的思路和常用命令的使用。掌握这些命令不仅能应对面试，在实际工作中排查网络问题也非常实用。本文覆盖面试中最高频出现的网络命令和工具。

## [](#ping命令)ping命令

### [](#基本用法)基本用法

ping是最基础的网络诊断工具，基于ICMP协议：

```bash
# 基本ping
ping www.baidu.com

# 指定次数
ping -c 4 www.baidu.com

# 指定包大小
ping -s 1024 www.baidu.com

# 设置超时时间（秒）
ping -W 3 www.baidu.com

# 快速模式，只等0.5秒
ping -i 0.5 www.baidu.com
```

### [](#ping的工作原理)ping的工作原理

```
客户端                                           服务器
   |                                               |
   |--- ICMP Echo Request (type=8, code=0) ------->|
   |                                               |
   |<-- ICMP Echo Reply (type=0, code=0) ----------|
   |                                               |
   |   RTT = 收到回复时间 - 发送请求时间            |
```

### [](#面试常问)面试常问

**Q: ping基于什么协议？端口号是多少？**

ping基于ICMP协议，ICMP是网络层协议，**没有端口号**。ICMP报文直接封装在IP数据报中，不经过运输层。

**Q: ping不通有哪些原因？**

- 目标主机宕机或网络不可达
- 防火墙/安全组禁用了ICMP
- 路由配置问题
- ARP解析失败
- 本地DNS解析失败

**Q: ping和traceroute的区别？**

- ping测试端到端的连通性
- traceroute追踪数据包经过的每一跳路由器

## [](#traceroute命令)traceroute命令

### [](#基本用法)基本用法

```bash
# Linux/macOS
traceroute www.baidu.com

# 指定探测包数量（每一跳发送的包数）
traceroute -q 3 www.baidu.com

# 不解析主机名，直接显示IP
traceroute -n www.baidu.com

# 设置最大跳数
traceroute -m 30 www.baidu.com

# Windows下使用tracert
tracert www.baidu.com
```

### [](#工作原理)工作原理

```
第1跳：TTL=1 → 到达第一个路由器 → TTL减为0 → 路由器返回 ICMP Time Exceeded
第2跳：TTL=2 → 到达第二个路由器 → TTL减为0 → 路由器返回 ICMP Time Exceeded
...
第N跳：TTL=N → 到达目标主机 → 目标返回 ICMP Echo Reply
```

- **利用TTL（生存时间）字段**逐跳探测
- 每次发送一个TTL递增的IP数据包
- 路由器将TTL减1，当TTL=0时返回ICMP超时报文
- 最后到达目标时返回ICMP回显应答

### [](#面试常问)面试常问

**Q: Linux traceroute和Windows tracert有什么区别？**

| 特性 | Linux traceroute | Windows tracert |
|------|-----------------|----------------|
| 默认协议 | UDP | ICMP |
| 端口 | 目标端口33434+ | — |
| 防火墙影响 | UDP可能被过滤 | ICMP可能被过滤 |
| 加-n参数 | 不解析域名 | 不解析域名 |

Linux默认使用UDP报文，Windows默认使用ICMP报文。Linux也可用 `traceroute -I` 使用ICMP模式。

## [](#netstat命令)netstat命令

### [](#基本用法)基本用法

```bash
# 查看所有TCP连接
netstat -t

# 查看所有监听端口
netstat -tlnp

# 查看所有UDP连接
netstat -ulnp

# 查看所有套接字
netstat -an

# 查看指定端口
netstat -tlnp | grep 8080

# 查看网络统计信息
netstat -s

# 查看路由表
netstat -rn
```

### [](#常用参数说明)常用参数说明

| 参数 | 含义 |
|------|------|
| -t | TCP连接 |
| -u | UDP连接 |
| -l | 仅显示监听(LISTEN)状态的套接字 |
| -n | 以数字形式显示地址和端口（不解析域名） |
| -p | 显示进程PID和名称 |
| -a | 显示所有套接字（包括LISTEN） |
| -r | 显示路由表 |
| -s | 显示网络统计 |

### [](#面试常问)面试常问

**Q: TIME_WAIT和CLOSE_WAIT状态是什么？**

```
                TIME_WAIT                        TIME_WAIT
主动关闭方: FIN-1 → FIN_WAIT_1 → FIN_WAIT_2 → TIME_WAIT → CLOSED
被动关闭方: CLOSE_WAIT ← CLOSE_WAIT ← ESTABLISHED ← ESTABLISHED
              ↓
            LAST_ACK
              ↓
           CLOSED
```

- **TIME_WAIT**：主动关闭方收到对方的FIN后进入此状态，持续**2MSL**（最大报文生存时间，约60秒）
  - 确保最后的ACK能到达对方
  - 确保网络中的延迟报文过期

- **CLOSE_WAIT**：被动关闭方收到FIN但未发送自己的FIN
  - 表示对方已关闭连接，但本方尚未关闭
  - 通常是因为代码中没有正确关闭连接（资源泄漏）

**Q: 大量TIME_WAIT怎么办？**

- 调整 `tcp_tw_reuse = 1` 允许将TIME_WAIT连接重新用于新连接
- 调整 `tcp_tw_recycle = 1`（不推荐，可能导致NAT问题）
- 使用长连接减少频繁建立/断开连接
- 使用SO_LINGER选项设置close行为

## [](#ss命令)ss命令

ss是netstat的现代替代品，速度更快：

```bash
# 查看所有TCP连接
ss -t -a

# 查看监听端口
ss -tlnp

# 查看TCP统计
ss -s

# 查看指定端口的连接
ss -tlnp | grep :8080

# 查看所有UDP连接
ss -ulnp

# 按状态过滤
ss -t state established
ss -t state time-wait
```

### [](#ss-vs-netstat)ss vs netstat

| 特性 | ss | netstat |
|------|-----|---------|
| 速度 | 快（直接读取内核数据） | 慢（遍历/proc） |
| 信息量 | 更多 | 较少 |
| 过滤功能 | 强大（支持状态过滤） | 基础 |
| 推荐度 | ⭐推荐 | 逐渐被淘汰 |

## [](#tcpdump命令)tcpdump命令

### [](#基本用法)基本用法

```bash
# 抓取所有网卡流量
tcpdump -i any

# 抓取指定网卡
tcpdump -i eth0

# 抓取指定主机
tcpdump host 192.168.1.100

# 抓取指定端口
tcpdump port 80

# 抓取指定网段
tcpdump net 192.168.1.0/24

# 抓取TCP包
tcpdump tcp

# 抓取HTTP请求
tcpdump -i any port 80 -A

# 保存到文件
tcpdump -i any -w capture.pcap

# 读取抓包文件
tcpdump -r capture.pcap

# 抓取前100个包
tcpdump -c 100

# 显示详细信息
tcpdump -vv
```

### [](#常用过滤表达式)常用过滤表达式

```bash
# 抓取源IP为指定地址的包
tcpdump src host 192.168.1.100

# 抓取目的IP为指定地址的包
tcpdump dst host 192.168.1.100

# 抓取指定端口范围的包
tcpdump portrange 80-90

# 组合条件
tcpdump 'src host 192.168.1.100 and dst port 80'

# 抓取SYN包（TCP三次握手的第一次）
tcpdump 'tcp[tcpflags] & (tcp-syn) != 0'

# 抓取HTTP GET请求
tcpdump -i any port 80 -A | grep "GET"
```

### [](#面试常问)面试常问

**Q: tcpdump和Wireshark的区别？**

- **tcpdump**：命令行抓包工具，适合服务器环境，无GUI
- **Wireshark**：图形化抓包分析工具，功能强大，适合桌面环境
- tcpdump抓包可保存为pcap文件，用Wireshark分析

## [](#curl命令)curl命令

### [](#基本用法)基本用法

```bash
# GET请求
curl https://api.example.com/users

# POST请求（JSON）
curl -X POST https://api.example.com/users \
  -H "Content-Type: application/json" \
  -d '{"name":"zhangsan","age":25}'

# 查看响应头
curl -I https://www.baidu.com

# 查看完整响应（包括头和体）
curl -v https://www.baidu.com

# 下载文件
curl -O https://example.com/file.zip

# 跟随重定向
curl -L https://example.com

# 指定超时时间
curl --connect-timeout 5 --max-time 10 https://example.com

# 使用代理
curl -x http://proxy:8080 https://example.com

# 显示请求耗时
curl -o /dev/null -s -w "time_total: %{time_total}s\n" https://example.com

# 查看DNS解析时间
curl -o /dev/null -s -w "time_namelookup: %{time_namelookup}s\n" https://example.com

# 使用指定证书
curl --cacert ca.crt https://example.com

# 只查看HTTP状态码
curl -o /dev/null -s -w "%{http_code}\n" https://example.com
```

### [](#面试常问)面试常问

**Q: curl -v输出中的各阶段耗时怎么分析？**

```
time_namelookup  DNS解析耗时
time_connect     TCP连接耗时
time_appconnect  TLS握手耗时（HTTPS）
time_starttransfer  从请求到收到第一个字节的时间
time_total       总耗时
```

## [](#nslookupdig命令)nslookup/dig命令

### [](#nslookup)nslookup

```bash
# 查询域名
nslookup www.baidu.com

# 指定DNS服务器
nslookup www.baidu.com 8.8.8.8

# 反向查询
nslookup 8.8.8.8
```

### [](#dig)dig（更强大的DNS查询工具）

```bash
# 查询A记录
dig www.baidu.com

# 查询MX记录（邮件记录）
dig mx gmail.com

# 查询NS记录
dig ns baidu.com

# 查询CNAME记录
dig cname www.baidu.com

# 使用指定DNS服务器
dig @8.8.8.8 www.baidu.com

# 简短输出
dig +short www.baidu.com

# 查看完整的DNS解析过程
dig +trace www.baidu.com
```

### [](#面试常问)面试常问

**Q: DNS查询的过程是什么？**

```
浏览器缓存 → 操作系统缓存 → 本地DNS服务器 → 根DNS服务器
→ 顶级域DNS服务器 → 权威DNS服务器
```

使用 `dig +trace` 可以看到完整的解析路径。

## [](#arp命令)arp命令

```bash
# 查看ARP缓存表
arp -a

# 添加静态ARP条目
arp -s 192.168.1.1 00:11:22:33:44:55

# 删除ARP条目
arp -d 192.168.1.1

# Linux下使用ip命令查看
ip neigh show
```

## [](#ip命令)ip命令

ip命令是ifconfig的现代替代品：

```bash
# 查看网络接口
ip addr show

# 查看路由表
ip route show

# 添加默认网关
ip route add default via 192.168.1.1

# 查看ARP表
ip neigh show

# 启用/禁用网卡
ip link set eth0 up
ip link set eth0 down
```

## [](#telnetnc命令)telnet/nc命令

### [](#telnet)telnet

```bash
# 测试端口连通性
telnet 192.168.1.100 8080

# 测试HTTP服务
telnet www.baidu.com 80
GET / HTTP/1.1

```

### [](#ncnetcat)nc（netcat）

```bash
# 测试端口连通性
nc -zv 192.168.1.100 8080

# 扫描端口范围
nc -zv 192.168.1.100 80-90

# 简易HTTP服务器
nc -l 8080

# 文件传输
# 接收端
nc -l 8080 > file.txt
# 发送端
nc 192.168.1.100 8080 < file.txt
```

## [](#lsof命令)lsof命令

```bash
# 查看指定端口被哪个进程占用
lsof -i :8080

# 查看指定进程打开的文件
lsof -p 12345

# 查看所有网络连接
lsof -i

# 查看指定用户的所有连接
lsof -u username -i

# 查看所有监听的TCP端口
lsof -i TCP -s TCP:LISTEN
```

## [](#网络排错思路)网络排错思路

面试中经常被问"服务器无法访问怎么排查"，可以按照**OSI模型从下往上**逐层排查：

### [](#排错流程)排错流程

```
1. 物理层：网线是否插好？网卡指示灯是否亮？
   ↓
2. 数据链路层：ARP是否能解析？MAC地址是否正确？
   ↓ ip addr / arp -a
3. 网络层：IP是否配置正确？路由是否可达？
   ↓ ping / traceroute / ip route
4. 传输层：端口是否开放？防火墙是否拦截？
   ↓ telnet / nc -zv / ss -tlnp
5. 应用层：服务是否正常？DNS是否解析？HTTP状态码？
   ↓ curl / dig / nslookup
```

### [](#常见排查场景)常见排查场景

**场景1：ping不通目标服务器**

```bash
# 1. 检查本机网络
ping 127.0.0.1
ip addr show

# 2. 检查网关
ping 192.168.1.1

# 3. 检查DNS
nslookup www.baidu.com 8.8.8.8

# 4. 检查路由
traceroute www.baidu.com
```

**场景2：端口访问不了**

```bash
# 1. 检查端口是否监听
ss -tlnp | grep 8080

# 2. 检查防火墙
iptables -L -n

# 3. 检查连接
nc -zv 192.168.1.100 8080

# 4. 查看进程状态
lsof -i :8080
```

**场景3：网络慢**

```bash
# DNS解析时间
dig +stats www.baidu.com

# TCP连接时间
curl -o /dev/null -s -w "connect: %{time_connect}s\n" https://example.com

# 抓包分析
tcpdump -i any -w slow.pcap
```

## [](#总结)总结

网络命令和排错是面试中的实战考察点，需要重点掌握：

1. **ping**：ICMP协议，无端口号，测试端到端连通性
2. **traceroute**：利用TTL逐跳探测路由路径
3. **netstat/ss**：查看连接状态，TIME_WAIT/CLOSE_WAIT区别
4. **tcpdump**：命令行抓包，支持丰富的过滤表达式
5. **curl**：HTTP请求工具，可分析各阶段耗时
6. **dig/nslookup**：DNS查询，trace可查看完整解析链路
7. **lsof**：查看端口占用、进程网络连接
8. **排错思路**：从物理层到应用层逐层排查
