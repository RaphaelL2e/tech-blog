## 操作系统概述

### [](#操作系统的定义)操作系统的定义

操作系统（Operating System, OS）是**控制和管理整个计算机系统的硬件与软件资源**，合理地组织、调度计算机的工作与资源的分配，进而为用户和其他软件提供方便接口与环境的程序集合。操作系统是计算机系统中最基本的**系统软件**。

> **面试高频问**：操作系统是什么？它有什么作用？
> 回答要点：① 资源管理者 ② 用户与硬件之间的接口 ③ 扩充机器（虚拟机）

### [](#操作系统的四大特征)操作系统的四大特征

| 特征 | 说明 | 重要程度 |
|------|------|---------|
| **并发** | 指两个或多个事件在**同一时间间隔**内发生（区别于并行：同一时刻） | ⭐⭐⭐⭐⭐ |
| **共享** | 系统中的资源可供多个并发进程共同使用 | ⭐⭐⭐⭐⭐ |
| **虚拟** | 把物理上的一个实体变为逻辑上的多个对应物（时分复用/空分复用） | ⭐⭐⭐⭐ |
| **异步** | 进程以不可预知的速度向前推进（走走停停） | ⭐⭐⭐⭐ |

> **关键区分**：
> - **并发 vs 并行**：并发是宏观上同时，微观上交替；并行是真正的同时执行
> - **互斥共享 vs 同时共享**：互斥共享（如打印机）一段时间内只允许一个进程访问；同时共享（如磁盘文件）允许多个进程同时访问

### [](#操作系统的主要功能)操作系统的主要功能

1. **处理机管理**：进程控制、进程同步、进程通信、进程调度
2. **存储器管理**：内存分配与回收、内存保护、地址映射、内存扩充（虚拟内存）
3. **设备管理**：缓冲管理、设备分配、设备处理（驱动）
4. **文件管理**：文件存储空间管理、目录管理、文件读写与保护
5. **用户接口**：命令接口（联机/脱机）、程序接口（系统调用）、GUI

### [](#操作系统的发展历程)操作系统的发展历程

| 阶段 | 特点 | 缺点 |
|------|------|------|
| 单道批处理 | 自动性、顺序性、单道性 | 资源利用率低 |
| 多道批处理 | 多道、宏观并行微观串行 | 无交互能力、平均周转时间长 |
| 分时操作系统 | 同时性、独立性、及时性、交互性 | 不能处理紧急任务 |
| 实时操作系统 | 及时性、可靠性 | 资源利用率不是首要目标 |
| 网络操作系统 | 网络通信、资源共享 | — |
| 分布式操作系统 | 分布式处理、透明性 | — |

> **面试真题**：分时操作系统和实时操作系统有什么区别？
> - 分时OS追求**公平**，让每个用户都能得到及时响应
> - 实时OS追求**及时和可靠**，必须在严格时间限制内完成

### [](#系统调用)系统调用

系统调用是操作系统提供给应用程序使用的**接口**，是用户程序请求操作系统服务的唯一方式。

**系统调用的分类**：
- 设备管理
- 文件管理
- 进程控制
- 进程通信
- 内存管理

> **为什么需要系统调用？**
> 用户程序不能直接执行特权指令（如I/O操作），必须通过系统调用陷入内核态，由操作系统代为执行，从而**保护系统安全**。

**用户态 → 内核态的切换时机**：
1. 系统调用（主动）
2. 中断（外部设备触发）
3. 异常（如缺页中断、除零错误）

## 进程管理

### [](#进程的定义与特征)进程的定义与特征

**进程（Process）** 是**资源分配和调度的基本单位**，是程序的一次执行过程。

进程实体（进程映像）由三部分组成：
- **PCB**（进程控制块）—— 进程存在的唯一标志
- **程序段** —— 程序代码
- **数据段** —— 运行时数据

**进程的五大特征**：

| 特征 | 说明 |
|------|------|
| 动态性 | 进程是程序的一次执行过程，有生命周期 |
| 并发性 | 多个进程可在同一时间段内同时运行 |
| 独立性 | 进程是独立运行、独立获取资源的基本单位 |
| 异步性 | 进程按各自独立的、不可预知的速度推进 |
| 结构性 | 每个进程都有PCB + 程序段 + 数据段 |

> **进程 vs 程序**：
> - 程序是**静态**的，是指令的集合；进程是**动态**的，是程序的执行过程
> - 一个程序可以对应多个进程（如多次打开同一个应用）
> - 一个进程可以包含多个程序（主程序调用子程序）

### [](#进程的状态与转换)进程的状态与转换

**三状态模型**：

```
        调度              时间片到
 就绪 ──────→ 运行 ──────→ 就绪
  ↑              │
  │              │ 等待事件（I/O请求等）
  │              ↓
  └──── 阻塞 ←──┘
       （事件完成）
```

**五状态模型**（增加创建和终止）：
- **创建态**：分配资源、初始化PCB
- **就绪态**：已获得除CPU外的所有资源
- **运行态**：正在CPU上执行
- **阻塞态**：等待某一事件（I/O完成、资源分配等）
- **终止态**：回收资源、撤销PCB

> **面试高频问**：进程各状态之间能否两两转换？
> - 就绪 → 阻塞：**不可能**（未获得CPU，无法请求I/O）
> - 阻塞 → 运行：**不可能**（必须先到就绪态，等调度程序分配CPU）

**七状态模型**（增加就绪挂起和阻塞挂起）：

当内存不足时，进程可以被**挂起**（换出到外存）：
- **就绪挂起**：进程在外存，进入内存即可执行
- **阻塞挂起**：进程在外存，等待事件完成

### [](#进程控制块PCB)进程控制块（PCB）

PCB 是操作系统管理进程的**核心数据结构**，是进程存在的**唯一标志**。

**PCB 的主要信息**：

| 类别 | 包含内容 |
|------|---------|
| 进程标识符 | PID、父进程ID、用户ID |
| 处理机状态 | 通用寄存器、程序计数器PC、程序状态字PSW、栈指针 |
| 进程调度信息 | 进程状态、优先级、等待事件、调度策略 |
| 进程控制信息 | 程序和数据地址、资源清单、同步和通信机制 |

**PCB 的组织方式**：
- **线性表方式**：所有PCB放在一张线性表中，简单但查找效率低
- **链接方式**：按进程状态将PCB组成多个队列（就绪队列、阻塞队列等）
- **索引方式**：按进程状态建立几张索引表，通过索引查找PCB

### [](#进程控制)进程控制

进程控制由**原语**完成。原语是**不可中断**的原子操作，通过**关中断/开中断**实现。

**主要原语**：

1. **创建原语**：
   - 分配PID和PCB
   - 分配资源（内存、文件等）
   - 初始化PCB
   - 将PCB插入就绪队列

2. **撤销原语**：
   - 找到要撤销进程的PCB
   - 终止该进程及其所有子进程
   - 释放资源
   - 删除PCB

3. **阻塞原语**：
   - 保存运行现场
   - 将状态从运行改为阻塞
   - 插入等待队列
   - 转向进程调度

4. **唤醒原语**：
   - 从等待队列中移出PCB
   - 状态从阻塞改为就绪
   - 插入就绪队列

> **注意**：阻塞是进程**主动**的行为（自己请求I/O），唤醒是**被动**的行为（别的进程或系统唤醒它）。阻塞和唤醒必须**成对出现**。

### [](#进程调度)进程调度

**调度的三个层次**：

| 层次 | 说明 | 发生频率 |
|------|------|---------|
| 高级调度（作业调度） | 从外存的后备队列中选择作业调入内存 | 低（几分钟一次） |
| 中级调度（内存调度） | 将暂时不用的进程换出到外存（挂起） | 中等 |
| 低级调度（进程调度） | 从就绪队列中选择进程分配CPU | 高（毫秒级） |

> **只有低级调度是每个系统必须具备的**。

**进程调度的方式**：
- **非抢占式**：一旦分配CPU，直到进程完成或阻塞才释放
- **抢占式**：更高优先级的进程可以抢占当前进程的CPU

**调度算法**（⭐⭐⭐⭐⭐）：

| 算法 | 思想 | 优点 | 缺点 | 适用场景 |
|------|------|------|------|---------|
| FCFS（先来先服务） | 按到达顺序调度 | 公平、简单 | 短作业等待时间长 | 批处理系统 |
| SJF/SPF（短作业优先） | 选服务时间最短的 | 平均等待时间最短 | 长作业可能"饥饿" | 批处理系统 |
| SRTN（最短剩余时间优先） | 抢占式SJF | 平均周转时间最优 | 饥饿问题更严重 | 批处理系统 |
| 优先级调度 | 选优先级最高的 | 灵活 | 低优先级可能饥饿 | 实时系统 |
| 时间片轮转（RR） | 按时间片轮流执行 | 公平、响应快 | 时间片大小影响性能 | 分时系统 |
| 多级反馈队列 | 多个队列+时间片递增 | 兼顾各类进程 | 实现复杂 | 通用系统 |

> **面试必考**：SJF 的平均等待时间和平均周转时间在所有非抢占式算法中最短，但可能导致饥饿。
>
> **多级反馈队列的核心**：
> 1. 设置多个就绪队列，优先级从高到低，时间片从小到大
> 2. 新进程先进入第1级队列，用完时间片降级到下一级
> 3. 只有高优先级队列为空时，才调度低优先级队列
> 4. 高优先级进程可以抢占低优先级进程

**评价指标**：
- **周转时间** = 完成时间 − 到达时间
- **平均周转时间** = 所有进程周转时间之和 / 进程数
- **带权周转时间** = 周转时间 / 服务时间（≥1）
- **等待时间** = 周转时间 − 服务时间
- **响应比** = （等待时间 + 服务时间）/ 服务时间 = 1 + 等待时间/服务时间

## [](#经典面试题)经典面试题

### ⭐⭐⭐⭐⭐ 并发与并行的区别

- **并发（Concurrency）**：同一时间间隔内多个事件发生，宏观上同时，微观上交替执行（单核CPU上的多任务）
- **并行（Parallelism）**：同一时刻多个事件同时发生，需要多核/多CPU支持

### ⭐⭐⭐⭐⭐ 进程和程序的区别

| 维度 | 程序 | 进程 |
|------|------|------|
| 本质 | 静态的指令集合 | 动态的执行过程 |
| 生命周期 | 永久存在（直到删除） | 有生命周期（创建→执行→终止） |
| 资源 | 不占用CPU和内存资源 | 需要分配系统资源 |
| 对应关系 | 一个程序可对应多个进程 | 一个进程至少对应一个程序 |

### ⭐⭐⭐⭐ 为什么要引入进程？线程和进程有什么区别？

**引入进程的原因**：在多道程序环境下，程序不能独立运行，需要引入进程来描述程序的执行过程，实现并发执行。

**进程 vs 线程**：
- 进程是**资源分配**的基本单位，线程是**CPU调度**的基本单位
- 进程拥有独立的地址空间，同一进程的线程共享地址空间
- 进程切换开销大，线程切换开销小
- 进程间通信需要IPC机制，线程间可以直接读写进程数据段

### ⭐⭐⭐⭐ 什么是系统调用？与库函数有什么区别？

- **系统调用**：操作系统内核提供的接口，涉及用户态到内核态的切换
- **库函数**：语言或库提供的函数，运行在用户态，部分库函数会封装系统调用

区别：系统调用是内核级的，必须陷入内核态执行；库函数是用户级的，在用户态执行。
