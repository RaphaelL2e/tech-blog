# 计算机网络面试八股文（二）——数据链路层

## [](#数据链路层概述)数据链路层概述

### [](#数据链路层的基本功能)数据链路层的基本功能

数据链路层位于物理层和网络层之间，主要功能包括：

- **帧封装**：将网络层数据封装成帧，添加帧起始标志和校验序列
- **透明传输**：确保数据中的任何比特组合都能正确传输，不被误认为是控制信息
- **差错控制**：检测和纠正传输过程中产生的错误
- **流量控制**：协调发送方和接收方的数据传输速度
- **链路管理**：建立、维持和释放数据链路连接

### [](#组帧的方法)组帧的方法

1. **字符计数法**：在帧头部使用计数字段表示帧长度
2. **字符填充的首尾界符法**：使用特殊字符（如DLE、STX、ETX）标记帧的开始和结束
3. **比特填充的首尾标志法**：使用特殊的比特模式（如01111110）作为帧边界
4. **物理编码违例法**：使用无效的物理编码作为帧边界

**以太网使用比特填充法**：在数据部分遇到连续5个1时自动插入一个0

## [](#差错控制)差错控制

### [](#检错编码)检错编码

**奇偶校验码**：
- 奇校验：数据中1的个数为奇数
- 偶校验：数据中1的个数为偶数
- 只能检测奇数位错误，检错能力50%

**CRC循环冗余码**：
- 目前应用最广泛的检错码
- 原理：发送方和接收方约定一个生成多项式G(x)，数据除以G(x)得到余数R，将R作为CRC校验码
- 检错能力：能检测出所有突发长度≤r位的错误，以及大量随机错误
- **面试重点**：CRC不能100%检错，但实际应用中错误检出的概率非常高

### [](#纠错编码)纠错编码

**海明码**：
- 可以纠正单比特错误的纠错码
- 校验码位置：2的幂次方位（1、2、4、8...）
- 码距：任意两个合法码字之间不同比特位的最小数目
- 海明码的码距为3，可以检测2位错误或纠正1位错误

## [](#流量控制与可靠传输机制)流量控制与可靠传输机制

### [](#停止等待协议)停止-等待协议

最简单的流量控制协议：
- 发送方每发送一帧后必须等待接收方的确认（ACK）才能发送下一帧
- 接收方收到正确帧后发送ACK，收到错误帧后发送NAK
- 发送方设置超时计时器，超时未收到确认则重传

**信道利用率**：U = Td / (Td + RTT + Ta)

### [](#滑动窗口协议)滑动窗口协议

**后退N帧协议（GBN）**：
- 发送方维持一个大小为N的发送窗口
- 可以连续发送多个帧，但不需要等待确认
- 接收方按序接收，丢弃失序帧
- 如果帧i丢失，后续所有帧都会被重传

**选择重传协议（SR）**：
- 发送方维持大小为N的发送窗口
- 接收方可以缓存失序帧，只重传丢失的帧
- 每个帧都需要单独的确认
- 窗口大小通常 ≤ 2^(n-1)

**流水线协议**：
- 允许发送方连续发送多个帧而不等待确认
- 需要更大的序列号范围和更大的缓冲空间

## [](#介质访问控制)介质访问控制

### [](#信道划分)信道划分

将共享信道分割为多个子信道：

- **频分多路复用（FDM）**：按频率分割，适合模拟信号
- **时分多路复用（TDM）**：按时间片分割，分为同步TDM和统计TDM
- **波分多路复用（WDM）**：按波长分割，用于光纤通信
- **码分多路复用（CDM）**：所有用户同时使用相同频率，通过不同的编码区分

### [](#随机访问)随机访问

**ALOHA协议**：
- 纯ALOHA：用户随时发送数据，冲突后随机等待重发
- 时隙ALOHA：将时间划分为等长时隙，用户只能在时隙开始时发送

**CSMA协议**（载波监听多路访问）：
- 1-坚持CSMA：空闲立即发送，忙则持续监听
- 非坚持CSMA：空闲立即发送，忙则随机等待
- **P-坚持CSMA**：空闲时以概率P发送，忙则持续监听

**CSMA/CD协议**（载波监听多路访问/碰撞检测）：
- 用于总线型以太网
- 工作流程：先听后发、边听边发、冲突停发、随机重发
- **碰撞检测时间**：2τ（τ为端到端传播时延）
- **最小帧长**：确保在发送期间能检测到碰撞，以太网最小帧长为64字节
- **二进制指数退避算法**：冲突次数越多，等待时间越长

**CSMA/CA协议**（载波监听多路访问/碰撞避免）：
- 用于无线局域网（WLAN）
- 核心思想：尽量避免碰撞，而非检测碰撞
- 采用三种机制：预约信道、RTS/CTS帧、ACK确认

### [](#轮询访问)轮询访问

令牌环网和令牌总线网使用的介质访问控制方法：
- 只有持有令牌的节点才能发送数据
- 令牌在节点间循环传递
- 优点：无碰撞、公平
- 缺点：等待时间长、令牌开销

## [](#局域网)局域网

### [](#IEEE802标准)IEEE 802标准

- 802.3：以太网（CSMA/CD）
- 802.11：无线局域网（WLAN）
- 802.5：令牌环网
- 802.6：城域网
- 802.1：局域网标准体系结构、STP、Qos

### [](#以太网)以太网

**以太网特点**：
- 无连接、不可靠的服务
- 使用曼彻斯特编码
- 采用CSMA/CD协议
- 争用期：51.2μs（对于10Mbps以太网）
- 帧最小长度：64字节，最大长度：1518字节

**以太网帧格式**：

| 字段 | 长度 | 说明 |
|------|------|------|
| 前同步码 | 7B | 用于时钟同步 |
| 帧开始定界符 | 1B | 标志帧开始 |
| 目的MAC地址 | 6B | 目标主机地址 |
| 源MAC地址 | 6B | 源主机地址 |
| 类型/长度 | 2B | 上层协议类型 |
| 数据 | 46-1500B | IP数据报 |
| CRC校验 | 4B | 帧校验序列 |

### [](#MAC地址)MAC地址

- 48位（6字节）物理地址
- 前24位为厂商代码（OUI），后24位由厂商分配
- 最高位为I/G位：0表示单播，1表示组播
- 次高位为G/L位：0表示全球唯一，1表示本地分配

**ARP协议**：根据IP地址获取MAC地址
- 广播请求、单播响应
- 缓存ARP表，生存时间通常为20分钟

## [](#广域网)广域网

### [](#PPP协议)PPP协议

点对点协议（Point-to-Point Protocol）：
- 面向字节的协议
- 提供差错检测，但不提供纠错
- 支持多种网络层协议
- 支持多种链路类型
- 支持IP地址的动态分配

**PPP帧格式**：
- 标志字段（F）：01111110
- 地址字段（A）：11111111（固定）
- 控制字段（C）：00000011（固定）
- 协议字段：标识上层协议
- 数据部分：不超过1500字节
- FCS：帧校验序列

### [](#HDLC协议)HDLC协议

高级数据链路控制协议：
- 面向比特的协议
- 支持全双工通信
- 有三种帧类型：信息帧、监督帧、无编号帧
- 使用0比特插入法实现透明传输

## [](#数据链路层设备)数据链路层设备

### [](#网桥)网桥

- 工作在数据链路层
- 根据MAC地址转发帧
- 可以隔离碰撞域
- 具有学习功能，自动建立转发表

### [](#交换机)交换机

- 本质上是多端口网桥
- 每个端口独享带宽（不是共享）
- 工作模式：直通式、存储转发式、无碎片直通式
- 具有自学习功能，构建MAC地址表
- 支持VLAN划分

**交换机与集线器的区别**：

| 特性 | 集线器 | 交换机 |
|------|-------|--------|
| 工作层 | 物理层 | 数据链路层 |
| 转发方式 | 广播 | 基于MAC地址 |
| 带宽 | 共享带宽 | 独享带宽 |
| 冲突域 | 同一端口所有主机 | 每个端口独立 |
| 智能程度 | 无 | 学习MAC地址 |

## [](#总结)总结

数据链路层是计算机网络的核心层之一，需要掌握：

1. **组帧和透明传输**：理解各种组帧方法和透明传输原理
2. **差错控制**：CRC、海明码的原理和计算
3. **流量控制**：停止-等待、滑动窗口协议
4. **介质访问控制**：CSMA/CD、CSMA/CA的工作原理
5. **以太网**：帧格式、MAC地址、ARP协议
6. **交换机**：转发原理、与集线器的区别
