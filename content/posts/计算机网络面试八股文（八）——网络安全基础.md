## [](#网络安全概述)网络安全概述

### [](#网络安全的定义)网络安全的定义

网络安全是指保护计算机网络及其数据免受未经授权的访问、攻击、破坏或泄露。核心目标包括：

- **机密性（Confidentiality）**：确保信息只能被授权的人访问
- **完整性（Integrity）**：确保信息在传输过程中不被篡改
- **可用性（Availability）**：确保授权用户能够正常使用网络服务

### [](#常见的网络安全威胁)常见的网络安全威胁

| 威胁类型 | 攻击层 | 说明 |
|---------|-------|------|
| ARP欺骗 | 数据链路层 | 伪造ARP响应，将IP映射到攻击者MAC |
| DNS劫持 | 应用层 | 篡改DNS解析结果 |
| XSS | 应用层 | 跨站脚本攻击，注入恶意代码 |
| CSRF | 应用层 | 跨站请求伪造，诱导用户执行非预期操作 |
| SQL注入 | 应用层 | 在SQL查询中注入恶意代码 |
| DDoS | 网络层/传输层 | 分布式拒绝服务攻击 |
| 中间人攻击 | 多层 | 拦截和篡改通信内容 |

## [](#ARP欺骗)ARP欺骗

### [](#ARP协议回顾)ARP协议回顾

ARP（地址解析协议）用于将IP地址解析为MAC地址：

- 主机A需要主机B的MAC地址
- 主机A广播ARP请求："IP为192.168.1.2的MAC地址是多少？"
- 主机B单播ARP响应："192.168.1.2的MAC地址是BB:BB:BB:BB:BB:BB"

### [](#ARP欺骗原理)ARP欺骗原理

攻击者C发送伪造的ARP响应：

```
正常情况：   主机A认为 192.168.1.1的MAC是网关MAC
ARP欺骗后：  主机A认为 192.168.1.1的MAC是攻击者C的MAC
```

结果：主机A的所有上网流量都先经过攻击者C，再转发给网关

### [](#ARP欺骗的危害)ARP欺骗的危害

- **流量窃听**：攻击者可以监听所有经过的流量
- **会话劫持**：获取用户session或cookie
- **数据篡改**：修改传输中的数据
- **拒绝服务**：将流量导向不存在的位置

### [](#ARP欺骗的防御)ARP欺骗的防御

- **静态ARP绑定**：将正确的IP-MAC映射绑定到本地ARP表中
  ```bash
  arp -s 192.168.1.1 00:00:5e:00:01:01
  ```
- **使用ARP防火墙**：自动检测和阻止ARP欺骗
- **交换机端口安全**：配置802.1X端口认证
- **使用VLAN**：减小广播域，限制攻击范围

## [](#DNS劫持)DNS劫持

### [](#DNS工作原理回顾)DNS工作原理回顾

DNS（域名系统）将域名解析为IP地址：

1. 浏览器检查本地DNS缓存
2. 检查hosts文件
3. 查询本地DNS服务器
4. 递归查询根服务器、顶级域服务器、权威服务器
5. 返回IP地址

### [](#DNS劫持原理)DNS劫持原理

攻击者篡改DNS查询结果：

- **DNS污染**：在DNS响应到达用户之前进行篡改
- **DNS欺骗**：让用户使用攻击者控制的DNS服务器
- **Hosts文件篡改**：修改本地hosts文件

### [](#DNS劫持的危害)DNS劫持的危害

- **钓鱼网站**：将正常网站域名解析到恶意IP
- **流量劫持**：将用户流量引导到指定服务器
- **广告注入**：在网页中插入广告
- **恶意软件分发**：引导用户下载恶意软件

### [](#DNS劫持的防御)DNS劫持的防御

- **使用可信的DNS服务器**：如8.8.8.8（Google）、1.1.1.1（Cloudflare）
- **启用DNSSEC**：DNS安全扩展，验证DNS响应
- **定期检查hosts文件**：确保没有被篡改
- **HTTPS强制使用**：即使DNS被劫持，证书也能发现异常

## [](#XSS跨站脚本攻击)XSS跨站脚本攻击

### [](#XSS原理)XSS原理

跨站脚本攻击（Cross-Site Scripting）是将恶意脚本注入到网页中：

```html
<!-- 正常输入 -->
<input type="text" value="Hello">

<!-- 恶意输入 -->
<input type="text" value=""><script>alert('xss')</script><!">
```

当其他用户访问该页面时，恶意脚本会被执行。

### [](#XSS类型)XSS类型

**反射型XSS**：
- 恶意脚本作为用户请求参数，服务器直接返回
- 示例：搜索功能中，搜索词直接显示在页面

**存储型XSS**：
- 恶意脚本存储在服务器（数据库）中
- 所有访问该内容的用户都会受害
- 危害最大，如论坛帖子、评论区

**DOM型XSS**：
- 恶意脚本通过修改页面的DOM对象来执行
- 不涉及服务器端，纯客户端攻击

### [](#XSS的危害)XSS的危害

- **窃取Cookie**：获取用户的登录凭证
- **会话劫持**：冒充用户进行操作
- **键盘记录**：记录用户的键盘输入
- **网页篡改**：修改页面内容
- **植入恶意软件**：自动下载恶意软件

### [](#XSS的防御)XSS的防御

- **输入过滤**：过滤或转义用户输入的特殊字符（`<>`、`"`、`'`等）
- **输出编码**：在输出到HTML时进行编码（HTML实体编码）
- **设置CSP**：Content Security Policy，限制脚本来源
- **HttpOnly Cookie**：设置Cookie的HttpOnly属性，防止JavaScript访问

## [](#CSRF跨站请求伪造)CSRF跨站请求伪造

### [](#CSRF原理)CSRF原理

跨站请求伪造（Cross-Site Request Forgery）利用用户已登录的身份：

1. 用户登录银行网站，获取有效session
2. 用户访问恶意网站
3. 恶意网站包含向银行转账的请求
4. 浏览器自动携带Cookie发送请求
5. 银行验证Cookie后执行转账

### [](#CSRF的特点)CSRF的特点

- 攻击发生在第三方网站
- 利用用户已登录的身份
- 服务器认为请求是用户主动发起的
- 请求可以伪装成普通GET或POST请求

### [](#CSRF的防御)CSRF的防御

**方案1：验证码**

- 在关键操作（如转账）时要求用户输入验证码
- 用户体验较差

**方案2：Token验证**

- 服务器生成随机Token，嵌入表单或请求头
- 提交时验证Token的合法性
- 攻击者无法获取Token

**方案3：Referer检查**

- 检查HTTP请求的Referer头
- 验证请求来源是否是可信的网站
- 但Referer可以被伪造或为空

**方案4：SameSite Cookie**

- 设置Cookie的SameSite属性
- `SameSite=Strict`：完全禁止跨站Cookie
- `SameSite=Lax`：允许导航携带Cookie，但禁止AJAX/表单POST携带

## [](#SQL注入)SQL注入

### [](#SQL注入原理)SQL注入原理

用户输入中包含SQL语句，被服务器当作SQL代码执行：

```sql
-- 正常查询
SELECT * FROM users WHERE username='admin' AND password='123456'

-- 恶意输入（密码框输入：' OR '1'='1）
SELECT * FROM users WHERE username='admin' AND password='' OR '1'='1'
```

结果：绕过了密码验证

### [](#SQL注入类型)SQL注入类型

- **联合查询注入**：使用UNION获取其他表的数据
- **布尔盲注**：根据页面返回判断条件真假
- **时间盲注**：根据响应时间判断条件真假
- **报错注入**：利用数据库错误信息获取数据
- **堆叠查询**：执行多条SQL语句

### [](#SQL注入的防御)SQL注入的防御

- **使用参数化查询/预编译语句**：
  ```java
  // 使用 PreparedStatement
  String sql = "SELECT * FROM users WHERE id = ?";
  PreparedStatement stmt = conn.prepareStatement(sql);
  stmt.setInt(1, userId);
  ```
- **输入过滤**：严格验证用户输入的类型和格式
- **最小权限原则**：数据库账户只授予必要的权限
- **错误信息处理**：不向用户显示详细的数据库错误信息
- **使用ORM框架**：如Hibernate、MyBatis等

## [](#DDoS攻击)DDoS攻击

### [](#DDoS原理)DDoS原理

分布式拒绝服务攻击（Distributed Denial of Service）：

- 利用大量分布式的攻击源（僵尸网络）
- 向目标服务器发送大量请求
- 耗尽服务器资源，使其无法正常提供服务

### [](#DDoS攻击类型)DDoS攻击类型

**带宽消耗型**：
- **ICMP Flood**：发送大量ICMP数据包
- **UDP Flood**：发送大量UDP数据包
- **DNS Flood**：向DNS服务器发送大量查询请求

**资源消耗型**：
- **SYN Flood**：发送大量SYN包，不完成三次握手
- **HTTP Flood**：发送大量HTTP请求
- **慢连接攻击**：如Slowloris，保持连接不断开

### [](#DDoS的防御)DDoS的防御

- **增加带宽**：提高抗攻击能力
- **使用防火墙/IDS/IPS**：识别和过滤攻击流量
- **流量清洗**：使用云服务商的DDoS防护服务
- **CDN分发**：分散流量到多个节点
- **限制连接数**：配置服务器最大连接数
- **启用限流**：对异常请求进行限流

## [](#网络层防火墙)网络层防火墙

### [](#防火墙的概念)防火墙的概念

防火墙是位于内部网络和外部网络之间的安全设备：

- **包过滤防火墙**：检查每个数据包的头部信息（IP、端口、协议）
- **状态检测防火墙**：跟踪连接状态，更智能地过滤
- **应用层防火墙**：深度检查应用层数据

### [](#iptables基础)iptables基础

Linux下的iptables是最常用的防火墙工具：

```bash
# 查看规则
iptables -L -n

# 允许已建立的连接
iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT

# 允许特定端口
iptables -A INPUT -p tcp --dport 22 -j ACCEPT

# 拒绝其他所有入站
iptables -A INPUT -j DROP
```
