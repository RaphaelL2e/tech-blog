# 微服务安全深度解析

## 一、引言

从单体应用到微服务，安全架构面临全新挑战。单体时代，安全只需在入口处做一个登录认证；微服务时代，十几个服务如何传播用户身份？服务间调用如何认证？敏感数据如何保护？

本文将从微服务安全的挑战出发，深度解析认证（Authentication）、授权（Authorization）、网络安全、数据安全，以及Spring Security在微服务中的最佳实践。

### 单体 vs 微服务安全对比

```
单体应用安全
┌─────────────────────────────────┐
│        monolith.war             │
│  ┌─────────────────────────┐  │
│  │  登录认证（Session）    │  │
│  ├─────────────────────────┤  │
│  │  授权（RBAC）           │  │
│  └─────────────────────────┘  │
└─────────────────────────────────┘
优势：安全边界清晰（只有一个入口）
问题：无（单体应用的安全相对简单）

微服务安全
┌────┐  ┌────┐  ┌────┐  ┌────┐
│Gateway│──►│Service A│──►│Service B│──►│Service C│
└────┘  └────┘  └────┘  └────┘
挑战：
1. 用户身份如何在服务间传播？（Token传播）
2. 服务间调用如何认证？（mTLS/API Key）
3. 敏感数据如何保护？（加密/脱敏）
4. 如何实现统一授权？（集中式/分布式）
```

## 二、认证（Authentication）深度解析

### 2.1 JWT 深度解析

```java
// JWT 结构（Header.Payload.Signature）
// Header
{
  "alg": "RS256",  // 签名算法
  "typ": "JWT"
}

// Payload（载荷）
{
  "sub": "1234567890",  // Subject（用户ID）
  "name": "张三",
  "iat": 1516239022,    // Issued At（签发时间）
  "exp": 1516242622,    // Expiration（过期时间）
  "roles": ["USER", "ADMIN"],  // 自定义声明
  "scope": ["read", "write"]
}

// Signature（签名）
// RS256（RSA + SHA256）签名
byte[] signature = RSA.sign(
  base64UrlEncode(header) + "." + base64UrlEncode(payload),
  privateKey
);
```

**JWT 生成与验证流程**：

```
JWT 生成流程
┌────┐    ┌────────────┐    ┌────────────┐
│User  │──►│Auth Server │──►│ 返回JWT   │
└────┘    └────────────┘    └────────────┘
             │ 1.验证用户名密码
             │ 2.生成JWT（私钥签名）
             ▼
         JWT Token（eyJhbG...）

JWT 验证流程
┌────┐    ┌────────────┐    ┌────────────┐
│Client│──►│Resource   │──►│ 返回资源   │
│      │    │Server     │    │            │
└────┘    └────────────┘    └────────────┘
             │ 1.提取JWT
             │ 2.验证签名（公钥）
             │ 3.检查过期时间
             │ 4.提取用户信息
             ▼
         请求继续执行
```

### 2.2 JWT 最佳实践

```java
// 1. 使用RS256（非对称加密）而非HS256（对称加密）
@Configuration
public class JwtConfig {
    
    @Bean
    public JwtDecoder jwtDecoder() {
        // 使用公钥验证签名（RS256）
        Resource publicKey = new ClassPathResource("public.pem");
        return NimbusJwtDecoder.withPublicKey(readPublicKey(publicKey)).build();
    }
    
    @Bean
    public JwtEncoder jwtEncoder() {
        // 使用私钥签名（RS256）
        Resource privateKey = new ClassPathResource("private.pem");
        return NimbusJwtEncoder.withPrivateKey(readPrivateKey(privateKey)).build();
    }
}

// 2. 设置合理的过期时间（Access Token短，Refresh Token长）
public class TokenService {
    
    public String generateAccessToken(User user) {
        return Jwts.builder()
            .setSubject(user.getUserId())
            .claim("scope", user.getRoles())
            .setIssuedAt(new Date())
            .setExpiration(new Date(System.currentTimeMillis() + 900_000))  // 15分钟
            .signWith(privateKey, SignatureAlgorithm.RS256)
            .compact();
    }
    
    public String generateRefreshToken(User user) {
        return Jwts.builder()
            .setSubject(user.getUserId())
            .setIssuedAt(new Date())
            .setExpiration(new Date(System.currentTimeMillis() + 7 * 24 * 3600 * 1000L))  // 7天
            .signWith(privateKey, SignatureAlgorithm.RS256)
            .compact();
    }
}

// 3. 使用JTI（JWT ID）防止重放攻击
public class TokenService {
    
    public String generateToken(User user) {
        String jti = UUID.randomUUID().toString();  // 唯一ID
        
        // 存入Redis（防止重放）
        redisTemplate.opsForValue().set(
            "jti:" + jti, 
            "valid", 
            Duration.ofMinutes(15)
        );
        
        return Jwts.builder()
            .setId(jti)  // JTI
            .setSubject(user.getUserId())
            .setExpiration(new Date(System.currentTimeMillis() + 900_000))
            .signWith(privateKey, SignatureAlgorithm.RS256)
            .compact();
    }
    
    public boolean validateToken(String token) {
        String jti = Jwts.parserBuilder()
            .build()
            .parseClaimsJws(token)
            .getBody()
            .getId();
        
        // 检查JTI是否有效（未被撤销）
        return redisTemplate.hasKey("jti:" + jti);
    }
    
    public void revokeToken(String jti) {
        // 撤销Token（退出登录）
        redisTemplate.delete("jti:" + jti);
    }
}
```

### 2.4 Spring Security OAuth2 深度解析

```java
// 1. 资源服务器配置（验证JWT）
@Configuration
@EnableWebSecurity
public class ResourceServerConfig {
    
    @Bean
    public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
        http
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/public/**").permitAll()
                .requestMatchers("/admin/**").hasRole("ADMIN")
                .anyRequest().authenticated()
            )
            .oauth2ResourceServer(oauth2 -> oauth2
                .jwt(jwt -> jwt
                    .decoder(jwtDecoder())
                    .jwtAuthenticationConverter(jwtAuthenticationConverter())
                )
            );
        return http.build();
    }
    
    @Bean
    public JwtDecoder jwtDecoder() {
        // 从授权服务器获取公钥（JWK Set URL）
        return NimbusJwtDecoder.withJwkSetUri("http://auth-server:8080/oauth2/jwks").build();
    }
    
    @Bean
    public JwtAuthenticationConverter jwtAuthenticationConverter() {
        JwtGrantedAuthoritiesConverter authoritiesConverter = new JwtGrantedAuthoritiesConverter();
        authoritiesConverter.setAuthorityPrefix("ROLE_");
        authoritiesConverter.setAuthoritiesClaimName("roles");
        
        JwtAuthenticationConverter converter = new JwtAuthenticationConverter();
        converter.setJwtGrantedAuthoritiesConverter(authoritiesConverter);
        return converter;
    }
}

// 2. 客户端配置（获取Token）
@Configuration
public class OAuth2ClientConfig {
    
    @Bean
    public OAuth2AuthorizedClientManager authorizedClientManager(
        ClientRegistrationRepository clientRegistrationRepository,
        OAuth2AuthorizedClientService authorizedClientService
    ) {
        OAuth2AuthorizedClientProvider authorizedClientProvider =
            OAuth2AuthorizedClientProviderBuilder.builder()
                .clientCredentials()
                .build();
        
        DefaultOAuth2AuthorizedClientManager authorizedClientManager =
            new DefaultOAuth2AuthorizedClientManager(
                clientRegistrationRepository, 
                authorizedClientService
            );
        authorizedClientManager.setAuthorizedClientProvider(authorizedClientProvider);
        
        return authorizedClientManager;
    }
}

// 3. 使用Token调用下游服务
@Service
public class OrderService {
    
    private final OAuth2AuthorizedClientManager authorizedClientManager;
    
    public OrderDTO getOrderById(String orderId) {
        // 获取Token（客户端模式）
        OAuth2AuthorizedClient client = authorizedClientManager.authorize(
            OAuth2AuthorizeRequest.withClientRegistrationId("order-service")
                .principal("system")
                .build()
        );
        
        String token = client.getAccessToken().getTokenValue();
        
        // 使用Token调用下游服务
        return WebClient.builder()
            .baseUrl("http://order-service")
            .defaultHeader("Authorization", "Bearer " + token)
            .build()
            .get()
            .uri("/orders/{orderId}", orderId)
            .retrieve()
            .bodyToMono(OrderDTO.class)
            .block();
    }
}
```

### 3.1 RBAC（基于角色的访问控制）

```
RBAC 模型
┌───────────────────────────────────────────────┐
│  User（用户）                                │
│   ├── 张三（USER角色）                       │
│   └── 李四（ADMIN角色）                     │
└───────────────────┬─────────────────────────┘
                    │ 分配
                    ▼
┌───────────────────────────────────────────────┐
│  Role（角色）                                 │
│   ├── USER（用户角色）                       │
│   │   ├── 权限：read                         │
│   │   └── 权限：write（自己的资源）          │
│   └── ADMIN（管理员角色）                   │
│       ├── 权限：read                         │
│       ├── 权限：write                        │
│       └── 权限：delete                      │
└───────────────────┬─────────────────────────┘
                    │ 包含
                    ▼
┌───────────────────────────────────────────────┐
│  Permission（权限）                           │
│   ├── read（读取）                           │
│   ├── write（写入）                          │
│   └── delete（删除）                         │
└───────────────────────────────────────────────┘
```

```java
// RBAC 实现
@Entity
public class User {
    @Id
    private String userId;
    
    @ManyToMany
    private Set<Role> roles;
}

@Entity
public class Role {
    @Id
    private String roleId;
    
    @ManyToMany
    private Set<Permission> permissions;
}

@Entity
public class Permission {
    @Id
    private String permissionId;
    private String resource;  // 资源（如 /users/*）
    private String action;    // 操作（如 GET/POST）
}

// Spring Security 集成
@Service
public class CustomUserDetailsService implements UserDetailsService {
    
    @Override
    public UserDetails loadUserByUsername(String username) {
        User user = userRepository.findByUsername(username);
        
        // 提取用户的所有权限（角色+权限）
        Set<GrantedAuthority> authorities = new HashSet<>();
        for (Role role : user.getRoles()) {
            authorities.add(new SimpleGrantedAuthority("ROLE_" + role.getRoleId()));
            for (Permission permission : role.getPermissions()) {
                authorities.add(new SimpleGrantedAuthority(permission.getPermissionId()));
            }
        }
        
        return new org.springframework.security.core.userdetails.User(
            user.getUsername(),
            user.getPassword(),
            authorities
        );
    }
}
```

### 3.2 ABAC（基于属性的访问控制）

```
ABAC 模型
┌───────────────────────────────────────────────┐
│  Subject（主体）                              │
│   - 用户ID、角色、部门、清除级               │
├───────────────────────────────────────────────┤
│  Resource（资源）                             │
│   - 资源ID、类型、清除级、所有者             │
├───────────────────────────────────────────────┤
│  Action（操作）                               │
│   - 读取、写入、删除                         │
├───────────────────────────────────────────────┤
│  Environment（环境）                          │
│   - 时间、地点、设备、威胁级别               │
├───────────────────────────────────────────────┤
│  Policy（策略）                               │
│   - IF 主体.角色 == "ADMIN"                 │
│     AND 环境.时间 IN 工作时间               │
│     THEN 允许操作                             │
└───────────────────────────────────────────────┘
```

```java
// ABAC 实现（使用Spring Security + SpEL）
@Configuration
@EnableMethodSecurity
public class AbacConfig {
    
    @Bean
    public MethodSecurityExpressionHandler methodSecurityExpressionHandler() {
        DefaultMethodSecurityExpressionHandler handler = new DefaultMethodSecurityExpressionHandler();
        handler.setPermissionEvaluator(new AbacPermissionEvaluator());
        return handler;
    }
}

// 自定义PermissionEvaluator（ABAC策略引擎）
public class AbacPermissionEvaluator implements PermissionEvaluator {
    
    @Override
    public boolean hasPermission(Authentication authentication, Object targetDomainObject, Object permission) {
        User user = (User) authentication.getPrincipal();
        
        // 提取主体属性
        String userRole = user.getRole();
        String userDepartment = user.getDepartment();
        int userClearance = user.getClearance();
        
        // 提取资源属性
        Resource resource = (Resource) targetDomainObject;
        String resourceOwner = resource.getOwnerId();
        int resourceClearance = resource.getClearance();
        
        // 提取环境属性
        String currentTime = LocalTime.now().getHour() + "";
        String currentLocation = RequestContextHolder.currentRequestAttributes()
            .getSessionId();
        
        // ABAC策略评估
        return evaluatePolicy(userRole, userDepartment, userClearance,
                            resourceOwner, resourceClearance,
                            currentTime, currentLocation);
    }
    
    private boolean evaluatePolicy(String userRole, String userDepartment, 
                                  int userClearance, String resourceOwner,
                                  int resourceClearance, String currentTime,
                                  String currentLocation) {
        // 策略1：管理员可以访问所有资源
        if ("ADMIN".equals(userRole)) {
            return true;
        }
        
        // 策略2：同部门用户可以访问
        if (userDepartment.equals(resourceOwner)) {
            return true;
        }
        
        // 策略3：清除级足够可以访问
        if (userClearance >= resourceClearance) {
            return true;
        }
        
        // 策略4：工作时间外禁止访问敏感资源
        int hour = Integer.parseInt(currentTime);
        if (hour < 8 || hour > 18) {
            return false;  // 非工作时间
        }
        
        return false;
    }
}

// 使用注解（方法级授权）
@Service
public class DocumentService {
    
    @PreAuthorize("@abacPermissionEvaluator.hasPermission(authentication, #document, 'read')")
    public Document getDocumentById(String documentId, Authentication authentication) {
        Document document = documentRepository.findById(documentId);
        return document;
    }
}
```

### 3.3 OPA（Open Policy Agent）深度解析

```
OPA 架构
┌───────────────────────────────────────────────┐
│         Service（微服务）                     │
│   （需要授权决策）                           │
└───────────────────┬─────────────────────────┘
                    │ 1. 查询OPA（授权请求）
                    ▼
┌───────────────────────────────────────────────┐
│         OPA（策略引擎）                      │
│   ┌─────────────────────────────────┐      │
│   │  Policy（Rego语言）              │      │
│   │  ┌─────────────────────────┐    │      │
│   │  │ package authz             │    │      │
│   │  │                           │    │      │
│   │  │ allow {                   │    │      │
│   │  │   input.user.role == "admin" │    │      │
│   │  │ }                         │    │      │
│   │  │ deny {                    │    │      │
│   │  │   input.user.role != "admin" │    │      │
│   │  │ }                         │    │      │
│   │  └─────────────────────────┘    │      │
│   └─────────────────────────────────┘      │
│   ┌─────────────────────────────────┐      │
│   │  Data（数据）                    │      │
│   │  - 用户属性                      │      │
│   │  - 资源属性                      │      │
│   │  - 环境属性                      │      │
│   └─────────────────────────────────┘      │
└───────────────────┬─────────────────────────┘
                    │ 2. 返回决策（allow/deny）
                    ▼
┌───────────────────────────────────────────────┐
│         Service（微服务）                     │
│   （根据决策允许/拒绝请求）                   │
└───────────────────────────────────────────────┘
```

```yaml
# OPA Policy（Rego语言）
package authz

# 允许规则
allow {
    # 输入：用户、资源、操作
    input.user.role == "admin"
}

allow {
    input.user.id == input.resource.owner_id
}

allow {
    input.action == "read"
    input.user.department == input.resource.department
}

# 拒绝规则（默认拒绝）
deny {
    not allow
}
```

```java
// Spring Boot 集成 OPA
@Service
public class OpaAuthorizationService {
    
    private final WebClient webClient;
    
    public OpaAuthorizationService() {
        this.webClient = WebClient.builder()
            .baseUrl("http://opa:8181/v1/data/authz")
            .build();
    }
    
    public boolean authorize(User user, Resource resource, String action) {
        // 构造OPA输入
        Map<String, Object> input = new HashMap<>();
        input.put("user", Map.of(
            "id", user.getUserId(),
            "role", user.getRole(),
            "department", user.getDepartment()
        ));
        input.put("resource", Map.of(
            "owner_id", resource.getOwnerId(),
            "department", resource.getDepartment()
        ));
        input.put("action", action);
        
        // 查询OPA
        Map<String, Object> response = webClient.post()
            .bodyValue(Map.of("input", input))
            .retrieve()
            .bodyToMono(Map.class)
            .block();
        
        // 解析决策
        Boolean allow = (Boolean) ((Map) response.get("result")).get("allow");
        return allow != null && allow;
    }
}
```

### 4.2 服务间认证（Service-to-Service Authentication）

```java
// 1. API Key 认证（简单场景）
@Configuration
public class ApiKeyConfig {
    
    @Bean
    public FilterRegistrationBean<ApiKeyFilter> apiKeyFilter() {
        FilterRegistrationBean<ApiKeyFilter> bean = new FilterRegistrationBean<>();
        bean.setFilter(new ApiKeyFilter());
        bean.addUrlPatterns("/api/*");
        return bean;
    }
}

public class ApiKeyFilter implements Filter {
    
    @Override
    public void doFilter(ServletRequest request, ServletResponse response, FilterChain chain) 
        throws IOException, ServletException {
        
        HttpServletRequest httpRequest = (HttpServletRequest) request;
        String apiKey = httpRequest.getHeader("X-API-Key");
        
        // 验证API Key
        if (apiKey == null || !isValidApiKey(apiKey)) {
            HttpServletResponse httpResponse = (HttpServletResponse) response;
            httpResponse.setStatus(HttpServletResponse.SC_UNAUTHORIZED);
            httpResponse.getWriter().write("Invalid API Key");
            return;
        }
        
        chain.doFilter(request, response);
    }
    
    private boolean isValidApiKey(String apiKey) {
        // 从数据库或Redis验证API Key
        return apiKeyRepository.existsByApiKey(apiKey);
    }
}

// 2. mTLS认证（高安全场景）
@Configuration
public class MtlsConfig {
    
    @Bean
    public RestTemplate restTemplate() throws Exception {
        // 加载客户端证书
        KeyStore keyStore = KeyStore.getInstance("PKCS12");
        keyStore.load(new FileInputStream("/path/to/client.p12"), "password".toCharArray());
        
        // 加载信任库（服务器证书）
        KeyStore trustStore = KeyStore.getInstance("JKS");
        trustStore.load(new FileInputStream("/path/to/truststore.jks"), "password".toCharArray());
        
        // 配置SSLContext
        SSLContext sslContext = SSLContexts.custom()
            .loadKeyMaterial(keyStore, "password".toCharArray())
            .loadTrustMaterial(trustStore, null)
            .build();
        
        // 创建SSL连接工厂
        SSLConnectionSocketFactory socketFactory = new SSLConnectionSocketFactory(sslContext);
        
        return new RestTemplateBuilder()
            .requestFactory(() -> new HttpComponentsClientHttpRequestFactory(
                HttpClients.custom()
                    .setSSLSocketFactory(socketFactory)
                    .build()
            ))
            .build();
    }
}
```

```java
    }
    
    // 全局过滤器（Token传播）
    @Bean
    public GlobalFilter tokenPropagationFilter() {
        return (exchange, chain) -> {
            // 从请求中提取Token
            String token = exchange.getRequest()
                .getHeaders()
                .getFirst("Authorization");
            
            // 向下游服务传播Token
            ServerWebExchange mutatedExchange = exchange.mutate()
                .request(exchange.getRequest().mutate()
                    .header("Authorization", token)
                    .build())
                .build();
            
            return chain.filter(mutatedExchange);
        };
    }
```

### 5.2 数据脱敏

```java
// 数据脱敏（日志、API响应）
@Service
public class DataMaskingService {
    
    // 1. 邮箱脱敏
    public String maskEmail(String email) {
        if (email == null) return null;
        int atIndex = email.indexOf('@');
        if (atIndex <= 1) return "***@***.com";
        return email.charAt(0) + "***@" + email.substring(atIndex + 1);
    }
    
    // 2. 手机号脱敏
    public String maskPhone(String phone) {
        if (phone == null || phone.length() < 11) return "***";
        return phone.substring(0, 3) + "****" + phone.substring(7);
    }
    
    // 3. 身份证脱敏
    public String maskIdCard(String idCard) {
        if (idCard == null || idCard.length() < 15) return "***";
        return idCard.substring(0, 6) + "********" + idCard.substring(idCard.length() - 4);
    }
}

// 使用AOP自动脱敏（API响应）
@Aspect
@Component
public class DataMaskingAspect {
    
    @Around("@annotation(org.springframework.web.bind.annotation.GetMapping)")
    public Object maskSensitiveData(ProceedingJoinPoint joinPoint) throws Throwable {
        Object result = joinPoint.proceed();
        
        // 对返回结果进行脱敏
        if (result instanceof User) {
            User user = (User) result;
            user.setEmail(maskEmail(user.getEmail()));
            user.setPhone(maskPhone(user.getPhone()));
            user.setIdCard(maskIdCard(user.getIdCard()));
        }
        
        return result;
    }
}
```

```java
// 审计日志（记录所有敏感操作）
@Aspect
@Component
public class AuditLogAspect {
    
    @Autowired
    private AuditLogRepository auditLogRepository;
    
    @AfterReturning("@annotation(org.springframework.transaction.annotation.Transactional)")
    public void logAudit(JoinPoint joinPoint) {
        // 获取当前用户
        Authentication authentication = SecurityContextHolder.getContext().getAuthentication();
        String username = authentication.getName();
        
        // 获取操作方法
        MethodSignature signature = (MethodSignature) joinPoint.getSignature();
        Method method = signature.getMethod();
        
        // 记录审计日志
        AuditLog auditLog = new AuditLog();
        auditLog.setUsername(username);
        auditLog.setAction(method.getName());
        auditLog.setTimestamp(new Date());
        auditLog.setParameters(Arrays.toString(joinPoint.getArgs()));
        
        auditLogRepository.save(auditLog);
    }
}
```

## 六、面试高频问题

### Q1：OAuth2的四种授权模式？分别适用什么场景？

**答**：

| 模式 | 适用场景 | 特点 |
|------|----------|------|
| **授权码模式** | Web应用、移动APP | 最安全，通过授权码换Token |
| **隐式模式** | SPA（单页应用） | 已废弃，不安全 |
| **密码模式** | 信任的第一方应用 | 用户名密码直接换Token |
| **客户端模式** | 服务间调用 | 无用户参与，用Client ID+Secret换Token |

### Q2：JWT和Session的区别？

**答**：

| 维度 | JWT | Session |
|------|-----|---------|
| **状态** | 无状态（服务端不存储） | 有状态（服务端存储） |
| **扩展性** | 好（无状态，易扩展） | 差（需要Session共享） |
| **安全性** | 中（Token被盗用无法撤销） | 高（可以主动撤销） |
| **性能** | 好（无数据库查询） | 差（需要查询Session） |

### Q3：如何实现单点登录（SSO）？

**答**：使用OAuth2+OIDC（OpenID Connect）：
1. 用户访问Client应用
2. Client重定向到Auth Server（登录页）
3. 用户登录，Auth Server生成Id Token（用户信息）和Access Token
4. Client使用Access Token调用UserInfo端点获取用户信息
5. 后续访问其他Client时，使用同一个Id Token（SSO）

### Q4：mTLS的原理和适用场景？

**答**：mTLS（双向TLS）是客户端和服务器互相验证证书的TLS。
- **原理**：握手时，客户端和服务器都发送自己的证书，互相验证
- **适用场景**：服务间调用（Service Mesh）、金融系统、高安全要求场景
- **优势**：防中间人攻击、防重放攻击、强身份认证

### Q5：如何防止SQL注入？

**答**：
1. **使用预编译（PreparedStatement）**：参数化查询，避免SQL拼接
2. **使用ORM框架**：Hibernate/MyBatis自动转义
3. **输入验证**：白名单验证（只允许特定字符）
4. **最小权限原则**：数据库用户只给必要权限
5. **WAF（Web应用防火墙）**：检测SQL注入攻击

### Q6：如何保护API免受CSRF攻击？

**答**：
1. **CSRF Token**：每个表单包含随机Token，提交时验证
2. **SameSite Cookie**：设置Cookie的SameSite属性（Strict/Lax）
3. **验证Referer/Origin**：检查请求来源
4. **双重Cookie验证**：前端将Cookie中的Token放到Header中，后端验证

### Q7：微服务安全的最佳实践？

**答**：
1. **统一认证**：使用OAuth2/OIDC，所有服务信任同一个Auth Server
2. **Token传播**：使用Spring Cloud Sleuth或Istio，自动传播Token
3. **最小权限原则**：每个服务只给必要权限
4. **加密传输**：使用HTTPS/mTLS
5. **数据脱敏**：日志和API响应中脱敏敏感数据
6. **审计日志**：记录所有敏感操作
7. **定期安全扫描**：使用OWASP ZAP、Nessus等工具

## 七、总结

### 核心要点

1. **认证（Authentication）**：JWT（无状态、易扩展）、OAuth2（授权框架）、OIDC（身份认证）
2. **授权（Authorization）**：RBAC（基于角色）、ABAC（基于属性）、OPA（集中式策略引擎）
3. **网络安全**：mTLS（双向TLS）、服务间认证（API Key/mTLS）、API网关安全
4. **数据安全**：加密（AES/RSA）、脱敏（日志/API响应）、审计日志
5. **最佳实践**：统一认证、Token传播、最小权限、加密传输、数据脱敏、审计日志
