# 计算机网络面试八股文（七）——HTTPS与TLS/SSL

## [](#http存在的问题)HTTP存在的问题

HTTP协议存在严重的安全隐患：

- **明文传输**：数据在网络中明文传输，容易被窃听
- **不验证身份**：通信双方身份无法确认，可能遭遇中间人攻击
- **无法验证完整性**：数据可能被篡改，接收方无法发现

这些问题在实际生产环境中是致命的，尤其是涉及敏感信息（如密码、银行账户）时。

## [](#HTTPS的概念)HTTPS的概念

HTTPS（HyperText Transfer Protocol Secure）是HTTP的安全版本：

- **HTTPS = HTTP + SSL/TLS**
- 端口：HTTP使用80端口，HTTPS使用443端口
- 在HTTP和TCP之间增加SSL/TLS层，对数据进行加密
- 用户无感知，但数据得到了保护

### [](#HTTPS与HTTP的区别)HTTPS与HTTP的区别

| 特性 | HTTP | HTTPS |
|------|------|-------|
| 端口 | 80 | 443 |
| 安全性 | 明文传输，不安全 | 加密传输，安全 |
| 证书 | 不需要 | 需要CA证书 |
| osi层级 | 应用层 | 应用层（基于TLS） |
| 性能 | 较快 | 略慢（加密开销） |
| 成本 | 免费 | 需要购买SSL证书 |

## [](#加密技术基础)加密技术基础

### [](#对称加密)对称加密

加密和解密使用**相同的密钥**：

- **优点**：算法简单、速度快
- **缺点**：密钥传输不安全，密钥管理困难
- **代表算法**：AES、DES、3DES、RC4

**问题**：如何安全地将密钥传递给对方？

### [](#非对称加密)非对称加密

加密和解密使用**不同的密钥**（公钥和私钥）：

- **公钥**：公开给所有人
- **私钥**：自己保密
- 用公钥加密的数据只能用私钥解密
- 用私钥加密的数据只能用公钥解密
- **代表算法**：RSA、ECC（椭圆曲线加密）、DSA

**问题**：非对称加密速度慢，约为对称加密的100-1000倍

### [](#混合加密方案)混合加密方案

实际使用中采用两者结合的方式：

1. 通信开始时，使用**非对称加密**交换**会话密钥**
2. 后续数据传输使用**对称加密**（会话密钥）

既保证了密钥交换的安全性，又保证了数据传输的效率。

## [](#数字证书与CA)数字证书与CA

### [](#数字证书的作用)数字证书的作用

解决公钥归属问题——**如何确认公钥属于某个特定的人/机构**？

- 证书由**认证机构（CA）**颁发
- 证书中包含：公钥、持有者信息、颁发者信息、有效期、签名等
- 证书用CA的私钥签名，任何人都可以用CA的公钥验证

### [](#证书链)证书链

证书验证通常需要验证完整的证书链：

```
根证书（Root CA）
    └── 中间证书（Intermediate CA）
            └── 服务器证书（Server Certificate）
```

- 操作系统和浏览器内置了根证书
- 服务器需要发送完整的证书链供客户端验证

### [](#证书类型)证书类型

- **DV证书**（Domain Validation）：仅验证域名所有权，最便宜
- **OV证书**（Organization Validation）：验证组织身份，安全性更高
- **EV证书**（Extended Validation）：最严格的验证，浏览器地址栏显示绿色名称

## [](#TLSSSL握手过程)TLS/SSL握手过程

TLS握手是HTTPS通信的核心，它在传输数据之前建立安全的加密通道：

### [](#TLS1.2握手过程)TLS 1.2握手过程

```
Client                                              Server
   |                                                   |
   |--- ClientHello (支持的TLS版本、加密套件、随机数) -->|
   |                                                   |
   |<-- ServerHello (选定TLS版本、加密套件、随机数) ---|
   |<-- Certificate (服务器证书链) --------------------|
   |<-- ServerKeyExchange (DH算法参数) ---------------|
   |<-- CertificateRequest (请求客户端证书，可选) -----|
   |<-- ServerHelloDone -------------------------------|
   |                                                   |
   |--- Certificate (客户端证书，可选) ---------------->|
   |--- ClientKeyExchange (预主密钥，用公钥加密) ----->|
   |--- CertificateVerify (证书签名，可选) ----------->|
   |--- ChangeCipherSpec (开始使用加密) -------------->|
   |--- Finished (加密的握手消息摘要) ---------------->|
   |                                                   |
   |<-- ChangeCipherSpec (开始使用加密) --------------|
   |<-- Finished (加密的握手消息摘要) ----------------|
   |                                                   |
   |=========== 加密的HTTP数据通信 ===================|
```

### [](#TLS1.3的改进)TLS 1.3的改进

TLS 1.3对握手过程进行了大幅简化：

- **握手从2-RTT减少到1-RTT**（或0-RTT）
- 0-RTT：客户端可以直接在第一个请求中发送加密数据（基于之前会话的信息）
- 安全性更强，移除了不安全的加密算法
- 前向保密成为强制要求

### [](#前向保密)前向保密（Forward Secrecy）

- 即使服务器的私钥被泄露，过去的通信记录仍然安全
- 通过DH/ECDH临时密钥交换实现
- 每次会话使用不同的临时密钥对

## [](#中间人攻击)中间人攻击

### [](#什么是中间人攻击)什么是中间人攻击

攻击者插入到客户端和服务器之间：

```
Client <--攻击者伪造的证书--> 攻击者 <--真实连接--> Server
```

攻击者可以：
- 窃听所有通信
- 篡改数据
- 冒充客户端或服务器

### [](#如何防御中间人攻击)如何防御中间人攻击

- **证书校验**：客户端严格验证服务器证书的合法性
- **证书 pinning**：在应用代码中硬编码服务器的证书或公钥
- **安全意识**：用户不安装不受信的证书
- **HSTS**：HTTP Strict Transport Security，强制使用HTTPS

## [](#HTTP严格传输安全HSTS)HTTP严格传输安全（HSTS）

HSTS是一种安全策略机制：

```
Strict-Transport-Security: max-age=31536000; includeSubDomains
```

- 告诉浏览器在指定时间内只通过HTTPS访问该网站
- 防止HTTP降级攻击
- 自动将HTTP请求重写为HTTPS

## [](#HTTPS的优缺点)HTTPS的优缺点

### [](#优点)优点

- **数据加密**：防止窃听和篡改
- **身份验证**：通过证书确认服务器身份
- **SEO优化**：Google等搜索引擎优先收录HTTPS网站
- **避免劫持**：防止运营商插入广告

### [](#缺点)缺点

- **性能开销**：加密解密消耗CPU资源
- **延迟增加**：TLS握手需要额外的网络往返
- **证书成本**：商业证书需要费用（但有免费证书如Let's Encrypt）
- **复杂度增加**：证书管理、维护更复杂

## [](#实际应用中的注意事项)实际应用中的注意事项

### [](#SSL剥离攻击)SSL剥离攻击

攻击者阻止客户端使用HTTPS，迫使使用HTTP：

- 使用中间人攻击插入
- 防御：使用HSTS + 浏览器内置的预加载列表

### [](#TLS版本选择)TLS版本选择

- **TLS 1.0/1.1**：已不安全，应禁用
- **TLS 1.2**：目前主流，需要确保禁用不安全的加密套件
- **TLS 1.3**：推荐使用，更安全且性能更好

### [](#CDN与HTTPS)CDN与HTTPS

使用CDN加速时：
- 源站使用HTTP，CDN节点使用HTTPS
- CDN需要配置证书
- 最好全程使用HTTPS（源站也配置证书）

## [](#总结)总结

HTTPS与TLS/SSL是现代Web安全的基础，需要重点掌握：

1. **HTTP的安全问题**：明文传输、不验证身份、无法验证完整性
2. **HTTPS工作原理**：HTTP + TLS，使用混合加密
3. **对称加密vs非对称加密**：各自的优缺点和适用场景
4. **数字证书与CA**：解决公钥归属问题
5. **TLS握手过程**：1.2和1.3的区别
6. **前向保密**：DH/ECDH临时密钥
7. **中间人攻击与防御**：证书校验、证书 pinning、HSTS
