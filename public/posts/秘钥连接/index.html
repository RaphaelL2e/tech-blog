秘钥连接

2019-06-26

什么是linux的秘钥？

一个字符串对，分公钥和私钥

作用：

拥有私钥的客户端，可以直接访问拥有公钥的服务端。

一、秘钥的生成（一般在客户端生成秘钥对，将公钥存入服务端）

1 2 3 4 ssh-keygen Enter file in which to save the key (/your_home_path/.ssh/id_rsa): #直接回车 Enter passphrase (empty for no passphrase): [Type a passphrase] Enter same passphrase again: [Type passphrase again]

设置passphrase后，进行版本控制时，每次与GitHub通信都会要求输入passphrase，以避免某些“失误”

都可以直接回车跳过

二、将公钥存到服务端两种方法：

1 scp /root/.ssh/id_ras.pub root@xxx.xxx.xxx.xxx:/root/.ssh/authorized_keys

1 ssh-copy-id -i /root/.ssh/id_ras root@xxx.xxx.xxx.xxx

三、一些必要配置

1 2 3 4 5 6 7 8 9 vim /etc/ssh/sshd_config RSAAuthentication yes # 启用 RSA 认证，默认为yes PubkeyAuthentication yes # 启用公钥认证，默认为yes

AuthorizedKeysFile      .ssh/authorized_keys # 公钥认证文件，默认为.ssh/authorized_keys

另外，如果对服务器安全性比较高的情况下，可以设置用户只允许通过公钥认证，禁止用户用密码方式登录，只要修改一下服务器的配置文件/etc/sshd/sshd_config

PasswordAuthentication no

修改完后重启sshd

1 chmod 600 authorized_keys

四、连接

1 ssh -l root xxx.xxx.xxx.xxx
